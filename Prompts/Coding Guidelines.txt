* Modular design

* Use Next.js (App Router preferred)
* Use TypeScript
* Use Tailwind CSS for styling

* Each agent must:

  * Return structured JSON output
  * Include:

    * reasoning
    * confidence score
    * action recommendation

* No agent should directly modify UI state

5. Data & Communication Standards

* All inter-agent communication must be JSON-based
* Use strict schemas for:

  * Sensor data
  * Vision outputs
  * Planner decisions
* WebSocket messages must include:

  * event_type
  * timestamp
  * payload

* Prefer streaming updates over polling
* Sensor simulator must emit updates every 2–5 seconds
* UI must reactively update state

7. AI Integration Guidelines

* Use OpenAI API for:

  * Vision analysis
  * Planning decisions
  * Voice assistant
* Wrap all OpenAI calls in service layer
* Never call OpenAI directly from UI

8. Evaluation Guidelines

* Every agent output must be logged
* Track:

  * latency
  * confidence
  * cost estimation
* Build evaluation-first mindset

9. Hackathon Optimization Rules

* Prioritize working demo over completeness
* Use fallback/mock data if APIs fail
* Keep architecture simple and debuggable
* Avoid overengineering (no unnecessary microservices)

10. Code Quality Rules

* Write readable code over clever code
* Avoid deep nesting
* Use clear naming conventions
* Prefer functional components in frontend
* Keep backend services stateless where possible
